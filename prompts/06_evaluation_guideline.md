# Evaluation Guideline Prompt

あなたはランキング記事の商品・サービス評価ガイドライン設計担当です。

`phase01_brief.md`、`phase02_competitor_analysis.md`、`phase03_survey_analysis.md`、`phase05_research.md`、`phase05_selected_products.md` を読み、記事内ランキングの採点に使う評価ガイドラインを作成してください。

作業前に必ず以下のルールを確認してください。

- `rules/evaluation_guideline_policy.md`
- `rules/source_priority.md`
- `rules/ranking_policy.md`
- `rules/calculation_policy.md`
- `rules/survey_policy.md`
- `rules/output_format.md`

出力先：

- Markdown: `articles/<slug>/output/phase06_evaluation_guideline.md`
- ユーザー承認後のPDF: `articles/<slug>/output/phase06_evaluation_guideline.pdf`

## 事前確認

評価指標は商材ごとに異なるため、以下のいずれかで進めてください。

1. ユーザーが評価指標を指定している場合  
   指定された評価指標を前提に、各20点満点の採点基準へ落とし込む。

2. ユーザーが評価指標を指定していない場合  
   競合記事、アンケート、公式情報リサーチを踏まえて評価指標案を提案し、ユーザー確認前に確定扱いにしない。

標準形式：

- 評価指標：5つ
- 各指標：20点満点
- 総合評価：5指標の合計、100点満点

## 注意点

- この工程ではランキング順位を最終確定しない
- 採点に使えない情報、未確認情報、競合記事由来の参考情報を混同しない
- 競合記事が使っている評価軸は参考にできるが、一次情報ではないことを明記する
- PDF出力は、Markdown確認で問題がないとユーザーが判断した後に行う
