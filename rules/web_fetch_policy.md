# Web Fetch and Search API Policy

Web検索、Webフェッチ、検索API連携は、記事制作に必要なURL発見と情報確認のために使う。検索結果、スニペット、AI要約、競合記事の記述をそのまま本文根拠にしない。

## 利用目的

- 競合記事20件の候補収集
- 商品・サービス公式サイト、料金表、約款、重要事項説明、FAQ、キャンペーンページの発見
- 官公庁、公的機関、業界団体、統計資料、法令、白書などの外部リンク候補の発見
- 公式情報や一次情報ページの直接確認
- ファクトチェック時の根拠URL確認

## ツールの扱い

- Tavilyなどの検索APIは、候補URLの発見、検索順位の把握、関連ページ探索に使う
- Webフェッチは、対象URL本文、タイトル、更新日、料金表、条件、注意書き、PDFリンクなどの直接確認に使う
- APIレスポンスのスニペットやAI要約は補助情報であり、料金、契約条件、キャンペーン、スペックの根拠にしない
- Claude CLI二次チェックはレビュー用途であり、Web取得や公式確認の代替にしない

## 環境変数

- Tavily APIを使う場合は `TAVILY_API_KEY` を参照する
- APIキー値はログ、本文、成果物、プロンプトに出力しない
- APIキーが未設定または疎通できない場合は、利用不可として記録し、利用可能な検索・取得手段で代替する
- Claude Maxプラン運用を優先するため、Claude APIキーの利用は原則行わない

## 競合記事収集

- 指定キーワードと類似キーワードで検索し、上位表示される記事を収集する
- 同一ドメインの重複、商品・サービス公式サイト、明らかな販売ページは除外する
- 検索順位、検索キーワード、検索日時、検索手段を `competitor_targets.md` に残す
- 検索APIで得た順位やタイトルは、可能な範囲で対象URLを直接取得して確認する
- 公式ドメイン配下でも比較記事、レビュー記事、ランキング記事などのメディア記事は競合対象にできる

## 公式情報リサーチ

- 料金、キャンペーン、契約条件、解約条件、スペック、端末代、送料、違約金は公式サイトまたは公式PDFを最優先する
- 公式情報は、直接URL、確認日、対象ページ種別を `research.md` に残す
- PDF、約款、重要事項説明を確認した場合は、PDF URLと該当項目を記録する
- 公式ページで確認できない項目は `未確認` とする
- 競合記事や検索スニペットだけで確認した情報は、本文で断定しない

## 記録項目

Web取得を使った工程では、必要に応じて以下を残す。

- 検索クエリ
- 検索日時または確認日
- 使用手段（Tavily、通常Web検索、直接フェッチ、PDF取得など）
- 取得URL
- 情報源種別（公式、公式PDF、公的機関、業界団体、比較メディア、口コミなど）
- 本文に使えるかどうかの判定
- 未確認または追加確認が必要な項目

## 禁止事項

- APIキーを成果物へ出力しない
- 検索APIのスニペットだけで料金や条件を断定しない
- AI要約を一次情報として扱わない
- 取得日や出典URLのない数値を比較表やランキング根拠に使わない
- 古いキャッシュや更新日不明のページを最新情報として断定しない
